1. ความหมายของ Dead Code
Dead Code คือ โค้ดที่ถูกประมวลผล(Execute)แต่ผลลัพธ์ของมันไม่ได้ถูกนำไปใช้ต่อ หรือในความหมายกว้างๆ มักรวมถึงโค้ดที่ไม่มีทางถูกรันถึง(Unreachable Code)ด้วย
ผลเสีย คือ dead code จะทำให้ไฟล์มีขนาดใหญ่โดยไม่จำเป็น, สร้างความสับสนให้โปรแกรมเมอร์คนอื่น และเพิ่มภาระในการดูแลรักษา

2. ตัวอย่าง Code ที่เกิด Dead Code
function calculateTotal(price) {
  let discount = 10; // นี่คือ Dead Code เพราะค่าถูกเขียนทับทันที
  discount = 20; 
  
  let tax = price * 0.07;
  let result = (price - discount) + tax;
  
  return result;

  // ข้างล่างคือ Unreachable Code (เรียกใช้ไม่ได้เพราะติด return)
  console.log("Calculation finished"); 
  return 0;
}

3. ตัวอย่าง Code ที่ Refactoring แล้ว
function calculateTotal(price) {
  const discount = 20; // เก็บเฉพาะค่าที่ใช้งานจริง
  const tax = price * 0.07;
  
  return (price - discount) + tax;
}

4. กรณีที่ Dead Code ไม่ทำให้เกิดปัญหา หรือไม่จำเป็นต้องรีบแก้
ไม่ใช่ Dead Code :
  1. โค้ดที่เตรียมไว้สำหรับอนาคต (Placeholder): ฟังก์ชันที่เขียนดักไว้รอ API จริงเสร็จ แม้จะยังไม่มีใครเรียกใช้ แต่ถ้าลบทิ้งอาจต้องเสียเวลาเขียนใหม่ในวันพรุ่งนี้
  2. Logging/Debugging Code: โค้ดที่ใช้พ่น Log ออกมาดูตอนเจอบั๊ก (เช่น console.log) ในเครื่อง Developer แม้จะไม่ได้ใช้ตอน Production แต่ก็มีประโยชน์ตอนแก้ปัญหา
  3. Backward Compatibility: ใน Library หรือ SDK บางอย่าง เราอาจมีฟังก์ชันเก่าที่ไม่ได้ใช้งานแล้วในเวอร์ชันปัจจุบัน แต่ต้องเก็บไว้เพื่อให้โปรแกรมเก่าๆ ของลูกค้ายังทำงานได้
  4. ความเสี่ยงในการแก้สูงเกินไป: หากเป็นระบบเก่าแก่ (Legacy System) ที่ไม่มี Unit Test และโค้ดมีความซับซ้อนมาก การไปแตะ Dead Code เพียงบรรทัดเดียวอาจส่งผลกระทบลูกโซ่ (Side Effect) ที่คาดไม่ถึง การปล่อยไว้อาจปลอดภัยกว่าในระยะสั้น
